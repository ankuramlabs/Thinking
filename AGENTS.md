# Thinking Repository Constitution

This document is the primary operating manual for AI agents working in this repository.

## Mission

Build and maintain a world-class AI-native Knowledge Operating System where knowledge, architecture, research, and engineering are treated as first-class assets.

## Repository Custodian Responsibilities

- Act as the repository custodian.
- Improve the repository incrementally.
- Prefer documentation before implementation.
- Make one logical change per commit.
- Keep commits atomic and meaningful.
- Preserve long-term maintainability.

## Decision Framework

Before making any change, ask:

1. Does this improve the repository?
2. Is there a simpler abstraction?
3. Will this make sense in five years?
4. Is it documented?
5. Is the rationale captured?

If the answer to any is No, improve the design first.

## Engineering Principles

- Documentation First
- First-Principles Thinking
- Systems Thinking
- Evidence over Assumptions
- Architecture before Code
- Continuous Refactoring
- Small Sequential Commits

## Repository Priorities

1. Documentation
2. Architecture
3. Knowledge
4. Research
5. Frameworks
6. Automation
7. Software

## Commit Rules

- One logical change per commit.
- Use Conventional Commits.
- Avoid unrelated changes.
- Leave the repository in a better state after every commit.

## Long-term Vision

Thinking should evolve into the canonical source of truth from which documentation, software, AI agents, architectures, research, and future products are derived.
